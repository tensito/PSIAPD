# Отчет по лабораторной работе №2

## ПОСТАНОВКА ЗАДАЧИ, ЦЕЛЬ РАБОТЫ

Цель работы:  
Научиться реализовывать полный pipeline классификации видео на практике с использованием крупного видеодатасета HowTo100M, предобученной модели VideoMAE для извлечения признаков и классического алгоритма машинного обучения для итоговой классификации.

В рамках работы необходимо было решить задачу распознавания тематических категорий видеороликов, связанных с действиями человека и health-like тематикой. В качестве исходного источника данных использовался датасет HowTo100M, из которого была сформирована ограниченная подвыборка видео, подходящая для выполнения в среде Google Colab.

Постановка задач:

1. Загрузить metadata датасета HowTo100M.
2. Сформировать health-like подвыборку по ключевым словам из текстового описания и категорий видео.
3. Скачать доступные видеоролики по YouTube ID с помощью `yt-dlp`.
4. Выполнить предварительную обработку видео: извлечение кадров, изменение размера кадров и подготовку входных данных для модели.
5. Использовать предобученную модель VideoMAE как feature extractor.
6. Получить embedding-векторы видеороликов размерности 768.
7. Обучить классификатор Logistic Regression поверх извлечённых признаков.
8. Оценить качество классификации с помощью Accuracy, Precision, Recall, F1-score и confusion matrix.


## ТЕОРЕТИЧЕСКАЯ БАЗА

### Датасет HowTo100M

HowTo100M — это крупный видеодатасет, состоящий из обучающих видеороликов с YouTube. Он содержит большое количество видеоинструкций по различным тематикам и используется в задачах video understanding, video-text representation learning и video classification.

Полная версия HowTo100M слишком объёмная для обработки в Google Colab, поэтому в данной лабораторной работе использовалась не вся коллекция видео, а metadata датасета и малая тематическая подвыборка. Такой подход является практичным, так как позволяет использовать реальный источник данных, но не требует загрузки терабайтов видеоматериалов.

В работе из metadata выбирались записи, связанные с health-like тематикой. Для отбора использовались ключевые слова:

- `exercise`
- `workout`
- `yoga`
- `fitness`
- `massage`
- `meditation`
- `stretch`
- `training`
- `therapy`
- `rehab`
- `health`
- `body`
- `muscle`
- `gym`
- `pilates`
- `cardio`

На основе найденных слов видеоролики были отнесены к одной из категорий: `fitness`, `massage`, `stretch`, `therapy`. Эти категории использовались как классы для классификации.

### Предобработка видео

Так как нейронная сеть не принимает видеофайл напрямую, сначала из каждого ролика извлекались отдельные кадры. Для каждого видео выбиралось фиксированное число кадров, равномерно распределённых по всей длине ролика. Затем кадры приводились к единому размеру и передавались в предобученную модель.

Основные шаги предобработки:

1. Открытие видеофайла с помощью OpenCV.
2. Получение общего числа кадров.
3. Выбор 16 кадров по равномерной сетке.
4. Преобразование цветового пространства из BGR в RGB.
5. Изменение размера кадра.
6. Формирование списка кадров для подачи в VideoMAE.

Такой подход позволяет представить каждый видеоролик как последовательность кадров, которая затем обрабатывается моделью компьютерного зрения.

### Архитектура VideoMAE

VideoMAE — это модель для обработки видео, основанная на идее Vision Transformer и masked autoencoding. В отличие от классических сверточных сетей, VideoMAE работает не только с пространственной информацией внутри отдельных кадров, но и с временной структурой видеоролика.

В данной работе VideoMAE использовалась не как полностью обучаемая классификационная нейронная сеть, а как feature extractor. Это означает, что веса VideoMAE не дообучались на выбранной подвыборке. Модель применялась для преобразования каждого видео в числовой embedding-вектор.


Каждый видеоролик после обработки VideoMAE представлялся вектором размерности 768. Далее эти признаки использовались для обучения отдельного классификатора.


### Классификатор Logistic Regression

После извлечения признаков с помощью VideoMAE классификация выполнялась моделью Logistic Regression. Это линейный классификатор, который хорошо подходит для обучения поверх уже готовых embedding-векторов.

В данной работе Logistic Regression обучалась не на пикселях и не на кадрах напрямую, а на признаках, извлечённых предобученной нейронной сетью. Поэтому даже простая модель могла решать задачу многоклассовой классификации.

### Анализ результатов

Полученное значение accuracy составило 0.5556, то есть модель правильно классифицировала примерно 56% видеороликов на валидационной выборке. Для небольшой подвыборки HowTo100M такой результат можно считать приемлемым, так как задача классификации видео является более сложной, чем классификация изображений.

Лучший результат по F1-score был получен для класса `fitness`:

```text
F1-score = 0.67
```

Это означает, что модель лучше всего распознаёт видеоролики, связанные с тренировками, упражнениями и фитнесом. Вероятно, такие видео имеют более выраженные визуальные признаки: активные движения человека, спортивную обстановку, упражнения и характерные позы.

Класс `massage` показал recall 0.75, но precision 0.43. Это означает, что модель часто относит видео к классу `massage`, но часть таких предсказаний оказывается ошибочной. Иными словами, модель хорошо находит часть massage-видео, но иногда путает их с другими категориями.

Класс `stretch` показал сбалансированные значения precision и recall:

```text
Precision = 0.50
Recall = 0.50
F1-score = 0.50
```

Это говорит о том, что модель частично распознаёт stretching-видео, но из-за малого числа обучающих примеров и схожести движений с fitness или therapy категориями качество остаётся ограниченным.

Класс `therapy` имеет precision 0.67, но recall 0.40. Это значит, что когда модель предсказывает `therapy`, она достаточно часто оказывается права, однако значительная часть реальных therapy-видео классифицируется как другие классы. Причиной может быть размытость самой категории: therapy-видео визуально могут быть похожи на stretching, massage или rehabilitation exercises.


## ВЫВОДЫ ПО РАБОТЕ

В ходе выполнения лабораторной работы были решены все поставленные задачи:

1. Была загружена metadata датасета HowTo100M.
2. Была сформирована тематическая health-like подвыборка видеороликов.
3. Были скачаны доступные видео по YouTube ID с помощью `yt-dlp`.
4. Из каждого видеоролика были извлечены кадры.
5. Предобученная модель VideoMAE была использована для получения embedding-векторов видео.
6. Поверх признаков VideoMAE был обучен классификатор Logistic Regression.
7. Качество модели было оценено с помощью Accuracy, Precision, Recall и F1-score.

Основные выводы:

1. Использование предобученной VideoMAE позволяет решать задачу классификации видео даже при ограниченных вычислительных ресурсах. Вместо обучения тяжёлой нейросети с нуля модель использовалась как feature extractor, что значительно снизило нагрузку на Colab.
2. Logistic Regression показала работоспособность при классификации embedding-векторов, полученных из видео. Несмотря на простоту классификатора, модель достигла accuracy 0.5556 на валидационной выборке.
3. Качество модели ограничено размером подвыборки. Для повышения точности необходимо увеличить количество видеороликов, использовать больше классов и обеспечить более равномерное распределение примеров между категориями.
4. Автоматическая разметка по metadata является удобной, но не идеальной. Некоторые видео могут попадать в категорию по словам в описании, хотя визуально относиться к другой тематике.
5. Лучшие результаты модель показала для класса `fitness`, где F1-score составил 0.67. Это говорит о том, что визуальные признаки фитнес-видео наиболее хорошо отделяются от остальных категорий.
6. Классы `stretch` и `therapy` оказались сложнее для классификации, так как движения и визуальные сцены в этих категориях могут быть похожи друг на друга.

Цель работы достигнута: был реализован полный pipeline классификации видео на основе HowTo100M, VideoMAE и Logistic Regression. Полученные результаты подтверждают возможность использования предобученных видеомоделей для классификации реальных видеороликов при ограниченных вычислительных ресурсах.
